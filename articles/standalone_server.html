VISUAL STUDIO TEMPLATE

Download template & dlls from THIS link. Place the template as-is in “C:\Users\UserName\Documents\Visual Studio 2019\Templates\ProjectTemplates\Visual C#“ It will appears as follow in project creation window

Add/Update following references

  • FigNet.dll
  • FigNetProviders
  • FigNetServer
  • FigNetClient [if application type is client]

Set the provider of your choice & port number in ServerConfig.xml & run the project

Note

If you are using other IDE you can create project from scratch

CONSOLE BASED SERVER; create new .Netcore console application.

Add following references

  • FigNet.dll
  • FigNetProviders
  • FigNetServer
  • FigNetClient [if application type is client]

How to start a server Define ServerConfig tag in Configuration file

<ServerConfig>
    <Name>ServerNameHere</Name>
    <Port>5555</Port>
    <Provider>ENet</Provider>
</ServerConfig>

And set ApplicationType value to Server, It will Start IServerSocket that you can access via FN.Server property

<Configuration
  <ApplicationType>Server</ApplicationType>
  <FrameRate>33</FrameRate>
  <EnableFileLogs>true</EnableFileLogs>

How to start a Client Define PeerConfig tag under PeerConfigs in Configuration file

 <PeerConfigs>
    <PeerConfig>
      <Name>DefaultClient</Name>
      <Port>5555</Port>
      <PeerIp>127.0.0.1</PeerIp>
      <Provider>ENet</Provider>
      <AutoConnect>true</AutoConnect>
    </PeerConfig>
  </PeerConfigs>
Note
  • If AutoConnect is set to false then you have to manually start the client socket.
  • query PeerConfig from configuration: var peerConfig = Settings.PeerConfigs.Find(config=>config.Name == "DefaultClient");
  • and call this function FN.AddConnectionAndConnect(peerConfig);

And set ApplicationType value to Client, It will Start IClientSocket that you can access via FN.Connections[0] property

<Configuration
  <ApplicationType>Client</ApplicationType>
  <FrameRate>60</FrameRate>
  <EnableFileLogs>false</EnableFileLogs>
